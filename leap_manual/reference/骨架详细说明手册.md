# 骨架详细说明手册

> 来源：`leap-env/src/skeleton/`，`leap-vm/src/leapvm/skeleton/`（M03 全链路审查，2026-03-01）
> 更新：2026-03-01

---

## 第一部分：Skeleton 文件类型

Skeleton 系统有两类文件，均放在 `leap-env/src/skeleton/`：

| 类型 | 目录 | 命名规则 | 用途 |
|------|------|----------|------|
| type skeleton | `skeleton/type/` | `Xxx.type.skeleton.js` | 描述 Web API 类型（原型链 + 属性结构） |
| instance skeleton | `skeleton/instance/` | `xxx.instance.skeleton.js` | 描述全局单例对象（navigator / document / location 等） |

每个文件通过立即执行函数将一个对象 `push` 到 `leapenv.skeletonObjects[]`：

```js
(function(global) {
  const leapenv = global.leapenv || (global.leapenv = {});
  if (!Array.isArray(leapenv.skeletonObjects)) leapenv.skeletonObjects = [];
  leapenv.skeletonObjects.push({ /* skeleton 对象 */ });
})(globalThis);
```

## 第二部分：type skeleton 字段规范

### 顶层字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | `string` | 是 | 内部标识，**必须以 `.type` 结尾**（如 `"Navigator.type"`） |
| `ctorName` | `string` | 是 | V8 ClassName + 全局暴露的构造函数名（如 `"Navigator"`）；空字符串表示匿名 |
| `instanceName` | `string` | 否 | type skeleton 通常为空字符串；若填写会作为全局属性名安装单例 |
| `brand` | `string` | 是 | 品牌标签，用于 `Illegal invocation` 检查（如 `"Navigator"`） |
| `ctorIllegal` | `boolean` | 否 | `true` = `new Xxx()` 调用构造函数时抛 `TypeError`（默认 `false`） |
| `exposeCtor` | `boolean` | 否 | `true` = 把 `ctorName` 暴露到全局对象（默认 `true`） |
| `super` | `string \| null` | 是 | 父类 type skeleton 的 `name`（如 `"EventTarget.type"`）；`null` = 无继承 |
| `props` | `object` | 是 | 属性描述符映射，key 为属性名，value 为 PropertyDescriptor（见下） |

### PropertyDescriptor 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `owner` | `"constructor" \| "prototype" \| "instance"` | 属性挂载位置：构造函数自身 / 原型链 / 实例对象 |
| `kind` | `"data" \| "method" \| "accessor"` | 属性类型 |
| `brandCheck` | `boolean` | `true` = 调用时 C++ 侧验证 receiver 的内部字段品牌标签（防止 Illegal invocation） |
| `attributes` | `{ enumerable?, configurable?, writable? }` | V8 属性描述符标志（默认均为 `false`） |

**kind=data 专用字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `valueType` | `"string" \| "number" \| "boolean" \| "null" \| "undefined"` | 静态值的类型 |
| `value` | any | 静态数据值（构建期固化，不经过 dispatch） |

**kind=method 专用字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `length` | `number` | `Function.length` 属性值（形参个数），默认 0 |
| `dispatch` | `{ objName: string, propName: string }` | stub 触发后 dispatch 到 impl 的 key（`objName.propName`） |

**kind=accessor 专用字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `dispatch` | `{ getter?: { objName, propName }, setter?: { objName, propName } }` | getter/setter 各自的 dispatch key；只读属性可只填 getter |

### 典型示例（type skeleton）

```js
{
  "name": "Navigator.type",
  "ctorName": "Navigator",
  "instanceName": "",
  "brand": "Navigator",
  "ctorIllegal": true,
  "exposeCtor": true,
  "super": null,
  "props": {
    "userAgent": {
      "owner": "prototype",
      "kind": "accessor",
      "brandCheck": true,
      "attributes": { "enumerable": true, "configurable": true },
      "dispatch": {
        "getter": { "objName": "Navigator", "propName": "userAgent" }
      }
    },
    "sendBeacon": {
      "owner": "prototype",
      "kind": "method",
      "brandCheck": true,
      "length": 1,
      "dispatch": { "objName": "Navigator", "propName": "sendBeacon" }
    }
  }
}
```

## 第三部分：instance skeleton 字段规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | `string` | 是 | 内部标识，**必须以 `.instance` 结尾**（如 `"navigator.instance"`） |
| `instanceName` | `string` | 是 | 安装到全局对象的属性名（如 `"navigator"`），不可为空 |
| `brand` | `string` | 是 | 复用对应 type skeleton 的品牌标签（如 `"Navigator"`），支持跨帧品牌兼容检查 |
| `super` | `string \| null` | 是 | 继承自哪个 type skeleton 的 brand chain（如 `"Navigator"`）；决定实例对象的原型 |
| `ctorName` | `string` | 否 | 通常为空字符串 |
| `exposeCtor` | `boolean` | 否 | 通常为 `false`（单例不需要暴露构造函数） |
| `ctorIllegal` | `boolean` | 否 | 通常为 `false` |
| `props` | `object` | 是 | 通常为空 `{}`；特殊实例属性（不挂在原型上的直接属性）可在此定义 |

### 典型示例（instance skeleton）

```js
{
  "name": "navigator.instance",
  "instanceName": "navigator",
  "brand": "Navigator",
  "super": "Navigator",
  "ctorName": "",
  "exposeCtor": false,
  "ctorIllegal": false,
  "props": {}
}
```

## 第四部分：特殊字段与约束

### window.instance 的特殊性

`window.instance.skeleton.js` 是唯一一个 `instanceName = "window"` 的 instance skeleton，且必须**最先加载**（`generate-entry.js` 强制排第一位）。

`window.instance` 的 `props` 中定义了大量全局属性（`setTimeout`、`document`、`navigator` 等），并使用 `hide` / `placeholder` 标记控制 profile 过滤（由 `filterSkeletonObjectsForProfile` 在 `leapenv.loadSkeleton()` 时处理）。

### 继承与 super 字段

- type skeleton 的 `super` → C++ Phase 2 调用 `template->Inherit(parentTemplate)` 建立原型链
- instance skeleton 的 `super` → C++ 将实例对象的原型设置为对应 type 原型对象，确保 `instanceof` 检查正确

### globalThis 约束

所有 skeleton 文件的 IIFE 包装**必须使用 `})(globalThis);`**，不得使用 `})(this);`。构建时 `validate-skeleton-context.js` 扫描违规并使构建失败退出。原因：esbuild 打包为 IIFE 时，模块顶层 `this` 指向 `exports`（Node.js 模块上下文）而非全局对象。
