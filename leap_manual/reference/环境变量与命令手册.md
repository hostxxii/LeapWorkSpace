# 环境变量与命令手册

> 更新：2026-03-01（M02 填充：Hook 无生产 env var，记录测试专用变量）

---

## 第一部分：环境变量

### M01 — LeapVM 路径配置

| 变量 | 默认值 | 作用 | 范围 |
|------|--------|------|------|
| `LEAP_VM_PACKAGE_PATH` | 自动探测 `../leap-vm` | 指定 `leap-vm` 包入口路径 | `leap-env/runner.js` |
| `LEAP_VM_BINARY_PATH` | 自动探测 build 输出 | 指定 `leapvm.node` 路径 | `leap-vm/index.js` |

> 其余并发池、压测变量待对应模块（M09、M11 等）审查后填充。

### M04 — 运行时核心与配置

| 变量 | 默认值 | 作用 | 范围 |
|------|--------|------|------|
| `LEAP_DOM_BACKEND` | `dod` | DOM 后端选择：仅 `dod`（M06 已移除 `js`/`native`/`spec` 选项） | `runner.js` + `config.js` |
| `LEAP_SIGNATURE_PROFILE` | `fp-lean` | 签名容器 profile：`fp-lean`（默认）/ `fp-occupy` | `runner.js` + `config.js` |
| `LEAP_ENV_VERSION` | `1.0.0` | bundle 内环境版本号（写入 envDescriptor.envVersion） | `config.js` + `skeleton-loader.js` |
| `LEAPVM_HOST_LOG_LEVEL` | `info` | 宿主侧日志最低级别：`debug` / `info` / `warn` / `error` | `host-log.js` |
| `LEAP_TRACE` | `0` | 设为 `1` 开启每次 dispatch 调用的追踪日志（打印 JSON 到 console） | `runtime.js` |

**优先级说明：** `LEAP_DOM_BACKEND` 和 `LEAP_SIGNATURE_PROFILE` 由 runner.js 在 bundle 执行前通过 `applyDomBackendSetting` / `applySignatureProfileSetting` 将规范化值写入 `globalThis.__LEAP_DOM_BACKEND__` / `globalThis.__LEAP_SIGNATURE_PROFILE__`；bundle 内 `config.js` 优先读取这两个全局变量，环境变量作为兜底。

---

### M03 — Skeleton系统与Dispatch

本模块无生产 env var。Skeleton 加载行为通过 NAPI 调用（`defineEnvironmentSkeleton`）和配置对象（`leapenv.config.signatureProfile`、`leapenv.config.dispatchMissingMode`）控制，不读取 `LEAP_*` 环境变量。

`LEAP_ENV_VERSION`（由上层传入）被 skeleton-loader.js 用于 `envDescriptor.envVersion` 字段，详见 M04（运行时核心与配置）。

---

### M02 — Hook监控（测试专用变量）

| 变量 | 默认值 | 作用 | 范围 |
|------|--------|------|------|
| `LEAP_HOOK_ISO_TIMEOUT_MS` | `10000` | 控制 `test-leapenv-hook-isolation.js` 中 worker 线程等待超时（ms） | 仅测试脚本，生产代码无读取 |

> M02 生产代码（monitor/hook_filter/native_wrapper/builtin_wrapper）不读取任何 `LEAP_*` 环境变量；Hook 系统完全通过 NAPI 调用配置（`setMonitorEnabled` / `setPropertyBlacklist` / `installBuiltinWrappers`）。

---

### M05 — Window与全局对象模型

本模块无生产 env var。Window impl 的行为通过 `leapenv.signatureTaskState.windowMetrics`（由 siteProfile 注入，详见 M10）和 `global.__LEAP_HOST_TIMERS__`（由宿主在 bundle 执行前注入）控制，不读取 `LEAP_*` 环境变量。

---

### M06 — DOM-BOM实现层

本模块无新增生产 env var。DOM 后端选择由 `LEAP_DOM_BACKEND` 控制（已在 M04 记录）。impl 层行为通过 `leapenv.registerImpl` 注册和 dispatch 系统调用，不读取 `LEAP_*` 环境变量。

---

### M07 — DoD布局引擎

本模块无生产 env var。DoD 引擎通过 `module.exports` 暴露，调用方直接传参（`containerWidth`/`containerHeight`/`resetComputed`），不读取任何 `LEAP_*` 环境变量。

---

### M08 — iframe多上下文

本模块无生产 env var。子帧行为通过 `__createChildFrame__` 等 VM 内全局函数控制，`sameOrigin` 标志由 JS 侧 `isSameOrigin()` 动态计算（读取 `location.origin`），不读取 `LEAP_*` 环境变量。

---

### M09 — 并发池

| 变量 | 默认值 | 作用 | 范围 |
|------|--------|------|------|
| `LEAP_WORKER_CLEANUP_FAIL_LIMIT` | `3` | worker 内 DOM 清理失败累计上限；超过后 worker 在下次任务完成时回报 `shouldRecycle:true`，pool 主动 recycle 该 worker | `worker-common.js`（进程/线程 worker 共用） |

> **旧文档中提及的环境变量（`LEAP_POOL_MODE`、`LEAP_USE_PROCESS_POOL`、`LEAP_POOL_SIZE`、`LEAP_POOL_WORKER_MULTIPLIER`、`LEAP_TASK_TIMEOUT_MS` 等）均未在生产代码中实现读取**。这些参数通过 `new ThreadPool(options)` / `new ProcessPool(options)` 构造选项直接传入，由调用方控制，不走环境变量路径。

---

### M10 — 签名容器与配置注入

本模块无新增生产 env var。`LEAP_SIGNATURE_PROFILE` 已在 M04 记录。配置注入通过 `leapenv.applyFingerprintSnapshot` 等 JS API 完成，不读取 `LEAP_*` 环境变量。

---

### M11 — Inspector调试服务

| 变量 | 默认值 | 作用 | 范围 |
|------|--------|------|------|
| `LEAPVM_INSPECTOR_READY_WAIT_MS` | `200` | DevTools 连接后等待 `Debugger.enable` 在 VM 线程被实际处理的最长时间（ms）；设为 0 跳过等待，上限 5000ms | `leap_inspector_client.cc:ResolveInspectorReadyWaitMs()` |

> 增大此值可提高 `--inspect-brk` 可靠性；减小可降低 `waitForInspectorConnection` 体感延迟。默认 200ms 在当前调试场景下已足够。

---

### M12 — 构建与打包系统

本模块无生产 env var。构建脚本（`build.js`）通过命令行参数（`--watch`）控制行为，不读取 `LEAP_*` 环境变量。

---

### M13 — Intl/ICU方案

本模块无生产 env var。ICU 初始化在 `V8Platform::InitOnce()` 内硬编码调用 `v8::V8::InitializeICU()`，不读取 `LEAP_*` 环境变量。

## 第二部分：命令参考

> 来源：`通用命令查询手册.md`（2026-02-25 核对），迁入时间：2026-03-01

### 手工目标脚本调试（仓库根目录）

根目录无 `package.json`，统一使用以下手工入口：

| 命令 | 用途 | 类型 |
|------|------|------|
| `node run-work-leapvm.js --file h5st.js` | 在 LeapVM 中运行 `work/h5st.js` | run |
| `node run-work-leapvm.js --file h5st.js --debug` | 开启 Hook/JSHook 调试日志 | debug |
| `node run-work-leapvm.js --file h5st.js --site-profile site-profiles/jd.json` | 使用站点配置覆盖 API 返回值 | run |
| `node run-work-leapvm.js --file h5st.js --site-profile jd` | 按名称在 `site-profiles/` 查找（自动补 `.json`） | run |

常用参数：`--file`、`--debug`、`--site-profile`、`--dom-backend`。推荐站点配置目录：`site-profiles/`。

### leap-env（`leap-env/` 目录下执行）

| 命令 | 用途 | 类型 | 自动/人工 |
|------|------|------|-----------|
| `npm run build` | 构建环境 bundle | build | 自动 |
| `npm run dev` | 监听构建（watch 模式） | build/dev | 人工 |
| `npm run start` | 启动 `index.js` | run | 人工 |
| `npm run run` | 先构建再运行 | build/run | 人工 |
| `npm run debug` | 调试入口（`index_debug.js`） | debug | 人工 |
| `npm run check:skeleton` | 校验 skeleton 上下文（`})(globalThis)` 检查） | check | 自动 |
| `npm run check:entry` | 检查入口覆盖 | check | 自动 |
| `npm run gate:thread-mainline` | 线程池主线门禁脚本 | perf/gate | 自动 |
| `npm run bench:pool` | 进程池压测 | bench | 自动 |
| `npm run bench:thread-vs-process` | 线程池 vs 进程池对比压测 | bench | 自动 |
| `npm run bench:dom-final-shape-baseline` | DOM final shape 基线压测 | bench | 自动 |
| `npm run test:hook-isolation` | Hook 隔离测试 | test | 自动 |
| `npm run test:dom-memory-leak` | DOM 内存泄漏测试 | test | 自动 |
| `npm run test:dom-handle-guard` | DOM 句柄保护测试 | test | 自动 |
| `npm run test:dom-m2-minimal` | DOM M2 最小回归 | test | 自动 |
| `npm run test:dom-m3-minimal` | DOM M3 最小回归 | test | 自动 |
| `npm run test:dom-native-trace` | DOM native trace 测试 | test | 自动 |
| `npm run test:dom-native-ssot-consistency` | DOM native SSOT 一致性 | test | 自动 |
| `npm run test:dom-pool-isolation` | DOM pool 隔离测试 | test | 自动 |
| `npm run test:thread-pool-stability` | 线程池稳定性测试 | test | 自动 |

> `leap-env/package.json` 中 `test` 仍是占位命令（`Error: no test specified`），不要直接用 `npm test`。

### leap-vm 构建（CMake，`leap-vm/` 目录下执行）

`leap-vm` 构建不通过 `npm run` 编排，使用 CMake：

| 命令 | 用途 |
|------|------|
| `cmake -S leap-vm -B leap-vm/build -G "Visual Studio 17 2022" -A x64` | 生成工程文件 |
| `cmake --build leap-vm/build --config Release` | 编译 Release |

### 回归/压测 Runner（仓库根目录，PowerShell）

| 命令 | 用途 | 类型 | 自动/人工 |
|------|------|------|-----------|
| `& .\tests\runners\run-smoke.ps1` | 冒烟回归（核心快速验证） | test | 自动 |
| `& .\tests\runners\run-full.ps1` | 完整回归 | test | 自动 |
| `& .\tests\runners\run-perf.ps1` | 压测/门禁 | perf | 自动 |
| `& .\tests\runners\run-manual.ps1` | 手工验证项清单 | manual | 人工 |

**推荐顺序：** `run-smoke.ps1` → `run-full.ps1` → `run-perf.ps1` → （需要时）`run-manual.ps1`
