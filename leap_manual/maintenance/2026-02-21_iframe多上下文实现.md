## 背景

需要在 LeapVM 中为 `<iframe>` 提供浏览器风格的多上下文能力，以支持：

- `contentWindow`
- `contentDocument`
- `window.frames[n]`
- `window.length`

同时要求同源可访问、跨域安全降级、主/子帧 Skeleton 与 dispatch 隔离。

## 问题

在实现过程中暴露出三个关键问题：

1. 子 context 执行 bundle 时覆盖主 `skeleton_registry_`（registry 污染）
2. ChildFrame 结构入队时机过晚，导致 `NativeDefineEnvironmentSkeleton` 无法匹配子 context
3. `Window` skeleton 缺少 `length` dispatch route，`window.length` 为 `undefined`

## 修复 / 实现要点

### 1. SkeletonRegistry 可重放（Replay）

- `skeleton_registry.h/cc` 增加 `ReplaySkeletonsTo(...)`
- 深拷贝属性描述与 skeleton 对象
- 按 `skeleton_order_` 保持注册顺序一致

### 2. `VmInstance` 增加 ChildFrame 体系

- `ChildFrame` 保存：
  - `url`
  - `same_origin`
  - `context`
  - `registry`
  - `dispatch_fn`
- 新增：
  - 创建/导航/计数/proxy 原生回调
  - per-context dispatch 缓存方法
  - `FramesIndexedGetter`

### 3. 子帧创建流程重构（关键）

- 创建子 context
- 设置 SecurityToken（同源共享、跨域隔离）
- 安装 globals / console / timers / native wrapper
- 先登记 ChildFrame，再执行 bundle
- `NativeDefineEnvironmentSkeleton` 按当前 context 路由到对应 `cf.registry`

### 4. JS 侧 iframe 支持

- 新增 `HTMLIFrameElement.impl.js`
- 新增 `HTMLIFrameElement.type.skeleton.js`
- `Window.impl.js` 的 `length` getter 接入 `__getChildFrameCount__`
- `tools.js` 增加 `iframe -> HTMLIFrameElement` 标签映射
