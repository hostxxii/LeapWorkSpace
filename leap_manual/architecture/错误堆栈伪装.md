# 错误堆栈伪装（Error Stack Disguise）

> 特性编号：A2
> 源文件：`leap-env/src/core/runtime.js`（约 L969-L1035）

## 目的

在浏览器环境中，`Error.stack` 的格式和内容是 JS 代码可以观察到的指纹之一。LeapVM 作为嵌入式 V8 运行时，其原始堆栈帧会暴露 `leap-env`、`leap.bundle.js` 等内部文件路径，使被执行的脚本能够通过分析堆栈判断自身不在真实浏览器中运行。

错误堆栈伪装通过重写 `Error.prepareStackTrace`，将堆栈输出格式化为 **Chrome 风格**，同时 **过滤掉所有 leap 内部帧**，使 `Error.stack` 的结果与真实 Chrome 浏览器一致。

## 实现细节

### 入口

```js
Error.prepareStackTrace = function(error, structuredStack) { ... };
```

V8 在生成 `Error.stack` 字符串时会调用 `Error.prepareStackTrace`（若已定义），传入错误对象和 `CallSite` 结构化堆栈数组，由该函数返回最终的 `.stack` 字符串。

### 内部帧过滤规则

以下来源的帧会被静默丢弃，不出现在最终堆栈中：

| 过滤条件 | 说明 |
|----------|------|
| `fileName` 为 `null` | V8 内置帧 |
| `fileName` 以 `node:` 开头 | Node.js 内置模块帧 |
| `fileName` 包含 `leap-env` | leap-env 运行时帧 |
| `fileName` 包含 `leap.bundle.js` | 打包后的 leap 运行时 |
| `fileName` 包含 `bundle.js` | 通用打包帧 |
| `fileName` 包含 `entry.js` | 入口加载帧 |

过滤后只保留用户脚本自身的帧。

### Chrome 格式化

每一帧按 Chrome DevTools 的标准格式输出：

```
    at TypeName.funcName (filename.js:line:col)
    at filename.js:line:col
```

- 有 `typeName + funcName` 时输出 `TypeName.FuncName (location)`
- 有 `typeName + methodName`（无 funcName）时输出 `TypeName.MethodName (location)`
- 仅有 `funcName` 时输出 `FuncName (location)`
- 均无时直接输出 `location`

头部格式：`ErrorName: message`（与 Chrome 一致）。

### 再入守卫

```js
var _inPrepareStackTrace = false;
```

`prepareStackTrace` 内部访问 `CallSite` 方法（如 `getFileName()`）时，某些边界情况下可能触发新的 `Error` 构造，导致无限递归。通过 `_inPrepareStackTrace` 布尔守卫检测再入，再入时直接返回最小化的 `ErrorName: message` 字符串，避免递归崩溃。

### 异常容错

整个格式化逻辑包裹在 `try-catch-finally` 中：

- `catch`：如果 `CallSite` 方法调用失败（嵌入式 V8 中部分 CallSite API 可能不可用），退回到最小格式 `ErrorName: message`。
- `finally`：确保 `_inPrepareStackTrace` 复位。

## 与其他模块的关系

- **A1（ScriptOrigin）**：`prepareStackTrace` 中 `frame.getFileName()` 返回的路径依赖 V8 编译脚本时设置的 `ScriptOrigin`。A1 若正确配置了虚拟路径，A2 输出的文件名即为伪装后的路径。
- **I-8（LEAP_TRACE 追踪日志）**：两者同在 `runtime.js` 中但相互独立；LEAP_TRACE 追踪 dispatch 调用，A2 处理错误堆栈。
- **Skeleton / Dispatch**：dispatch 过程中抛出的异常会经过 A2 格式化，确保用户脚本的 `try-catch` 看到的堆栈是干净的。

## 注意事项

- 此重写在 `runtime.js` 加载时立即生效，作用于整个 V8 Isolate 生命周期。
- 如需调试 leap-env 内部问题，内部帧被过滤会增加排查难度；可临时注释 `Error.prepareStackTrace` 赋值或在过滤条件前加 `return` 以查看完整堆栈。
