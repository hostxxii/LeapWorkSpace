## 系统整体功能

LeapWorkSpace 是一套高保真浏览器自动化环境，用于在受控的独立 V8 沙盒中执行目标网站的签名脚本，同时模拟完整的 Web API 表面（DOM、BOM、Intl、Storage 等），以收集站点指纹或执行挑战验证逻辑。系统由两大子系统构成：**LeapVM**（C++ 层，管理 V8 Isolate 生命周期与底层桥接）和 **LeapEnv**（JS 层，构建完整 Web 运行时环境并执行目标脚本）。

## 端到端数据流

```
目标脚本（h5st.js / 签名脚本）
    │
    ▼
runner.js: executeSignatureTask(leapvm, task)
    │  ① siteProfile 注入（applyFingerprintSnapshot / applyStorageSnapshot / applyDocumentSnapshot）
    │  ② leapvm.runScript(targetScript)
    │
    ▼
LeapVM: VmInstance::RunScript
    │  V8 执行目标脚本
    │
    ▼
目标脚本访问 Web API（如 navigator.userAgent）
    │
    ▼
Skeleton stub（C++ FunctionTemplate / accessor getter）
    │
    ▼
DispatchBridge::StubCallback
    │  品牌校验（内部字段检查）→ 读取 DispatchMeta → 调用 VM 内 leapenv.__runtime.bridge.dispatch
    │
    ▼
dispatch（JS，runtime.js）
    │  在 leapenv.implRegistry 查找 Navigator.userAgent getter
    │
    ▼
NavigatorImpl.userAgent getter（leap-env/src/impl/Navigator.impl.js）
    │  读取 leapenv.signatureTaskState.navigator.userAgent
    │  → 返回注入值或默认值
    │
    ▼  （同时，若 MonitorEngine 开启）
HookFilter / MonitorEngine（monitor.cc）
    │  黑名单过滤 → 全覆盖模式记录属性访问日志
    │
    ▼
运行结果返回给 runner.js → executeSignatureTask 返回字符串结果
    │
    ▼
runner.js: resetSignatureTaskState()（清理任务态）
```

## 关键约束与边界

- **V8 隔离**：每个 `VmInstance` 拥有独立 V8 Isolate，不共享堆内存；多实例并发安全。
- **NAPI 边界**：GitNexus 静态分析无法跨越 JS→C++ 边界，`impact(upstream)` 对 NAPI 导出符号可能返回 0 调用方——属预期行为。
- **IIFE 包装**：`leap.bundle.js` 以 IIFE 格式打包，在 VM 内执行时无 `require`/`exports`，所有通信通过 `leapenv` 全局对象。
- **pool/ 不入 bundle**：`src/pool/` 目录是 Node.js 宿主代码，由 `generate-entry.js` 显式排除，不打包进 VM 环境。
