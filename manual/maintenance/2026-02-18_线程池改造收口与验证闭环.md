## 背景

该条时间线的核心目标是将线程池从“可运行”推进到“可作为主线维护对象”：

- 具备可配置的任务执行与超时控制
- 具备 worker 生命周期管理（回收 / 拉起 / 心跳监控）
- 具备统计接口，支持门禁脚本和压测脚本
- 在 `full` / `perf` 档位形成可重复执行的验证闭环

## 实现收口（当前可见能力）

### 1. 线程池基础执行模型

`ThreadPool` 提供：

- `start()`：预启动 worker 并建立心跳监控
- `runSignature()`：提交任务（支持 per-task timeout）
- `close()`：关闭池并等待 worker 退出（当前支持 `forceTerminate` 关闭路径）

关键配置项（构造参数）包含：

- `size`
- `taskTimeoutMs`
- `workerInitTimeoutMs`
- `maxTasksPerWorker`
- `heartbeatIntervalMs`
- `heartbeatTimeoutMs`

相关实现：

- `leap-env/src/pool/thread-pool.js`

### 2. 生命周期与稳定性机制

当前实现已包含以下收口能力（供主线运行/压测使用）：

- 心跳监控与超时强杀（heartbeat timeout）
- worker 回收与自动拉起（`recycled` / `respawned` 指标）
- 队列排空与 active task 管理
- 统计快照接口 `getStats()`（供门禁与压测输出）

这使线程池不再只是“吞吐实验实现”，而是具备主线健康检查与异常观测所需的运维面。

## 验证闭环（当前入口）

### 1. 功能稳定性（`full`）

- `tests/scripts/integration/test-leapenv-thread-pool-stability.js`

用途：

- 在并发条件下批量提交任务
- 统计失败数与吞吐
- 输出 `pool.getStats()` 快照

该脚本已纳入 root `full` 档位（通过 `tests/manifest/full.ps1` 编排）。

### 2. 主线健康门禁（`perf`）

- `tests/scripts/perf/check-thread-mainline-gate.js`

用途：

- 对线程池主线运行做自动门禁检查
- 校验项包括：
  - `failed <= maxFailed`
  - `successRatio >= minSuccessRatio`
  - `p99 <= maxP99Ms`
  - `rssGrowth <= maxRssGrowthMb`

该脚本是当前最直接的线程池健康兜底门禁入口。

### 3. 模式对比压测（`perf`）

- `tests/scripts/perf/bench-thread-vs-process.js`

用途：

- 在线程池 / 进程池同口径参数下对比吞吐与延迟
- 输出结构化 JSON 报告（`benchmarks/thread-vs-process-*.json`）
- 为后续阈值/趋势观察提供样本
