# 2026-02-23_回归结果JSON结构化摘要落地

日期：2026-02-23  
来源：`REPO_REORG_TODO.md` J3（后续可选评估项）

## 背景

根目录回归 runner 已输出：

- `summary.txt`
- `commands.log`
- `env.snapshot.txt`
- `perf-summary.md`（仅 perf）

这些文件适合人工阅读，但不利于后续自动聚合、趋势分析或 CI 消费。

## 决策

直接落地 JSON 结构化摘要，而不仅停留在评估：

- 在 `tests/runners/common.ps1` 中新增 `summary.json` 输出
- 所有 profile（`smoke/full/perf/manual`）统一生成
- 使用 `schemaVersion = 1`

## 输出内容（摘要）

`summary.json` 包含：

- 运行元数据（profile、repoRoot、runDir、开始/结束时间）
- 汇总计数（total/pass/fail/manual）
- 产物路径（`summary.txt`、`commands.log`、`env.snapshot.txt`、`perf-summary.md`）
- 失败项列表（名称 + stdout/stderr 路径）
- 每条命令结果（index/name/status/exitCode/duration/时间戳/命令行/日志路径）

## 实现位置

- `tests/runners/common.ps1`
  - `New-RunContext` 增加 `JsonSummary`
  - 新增 `Write-JsonSummary`
  - `Invoke-ManifestRun` / `Invoke-ManualRun` 在写 `summary.txt` 后统一写 JSON
- `tests/README.md`
  - 更新结果产物列表，加入 `summary.json`

## 兼容性说明

- PowerShell 5.1 可用（使用 `ConvertTo-Json`）
- 保留原 `summary.txt`，避免破坏现有人工查看流程
- `perf-summary.md` 仍只在 `perf` profile 生成，JSON 中通过 `artifacts.perfSummary` 表示（无则为 `null`）

## 验证建议

- 运行 `smoke` / `perf` / `manual`，确认各自结果目录均生成 `summary.json`
- 检查 `failures` 与 `results[]` 字段和 `summary.txt` 一致
