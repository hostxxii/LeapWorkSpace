# 环境变量与命令手册

本文档只记录当前 standalone 主线仍在使用的环境变量和常用命令。

## 1. 运行时环境变量

### Node / 宿主侧

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `LEAPVM_SERVER_PATH` | 自动探测 | `ServerManager` 查找 `leapvm_server` 二进制 |
| `LEAPVM_HOST_LOG_LEVEL` | `info` | 控制 `host-log.js` 输出级别 |
| `LEAP_ENV_VERSION` | `1.0.0` | bundle 内 `envDescriptor.envVersion` |
| `LEAP_DOM_BACKEND` | `dod` | 运行时 DOM backend 选择 |
| `LEAP_TRACE` | `0` | 打开 dispatch trace 日志 |
| `LEAP_PERF_TRACE` | `0` | 打开 runtime perf trace |
| `LEAP_PERF_DISPATCH_CACHE` | `0` | 打开 dispatch cache 统计 |

### 服务端 / C++ 侧

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `LEAPVM_LOG_LEVEL` | `info` | `leapvm_server` 日志级别 |
| `LEAPVM_CLI_SHOW_HOOK_LOGS` | `0` | 默认 CLI 隐藏 hook 噪声；设为真可显示 |
| `LEAPVM_INSPECTOR_READY_WAIT_MS` | `200` | DevTools 建连后等待 `Debugger.enable` 的上限 |
| `LEAPVM_ALLOW_DEVTOOLS_EVAL_HOOK_LOGS` | `0` | 允许 DevTools eval 触发 hook 日志 |
| `LEAPVM_TRACK_GC_OBJECT_STATS` | `0` | 给 V8 加 `--track-gc-object-stats` |
| `LEAPVM_PLATFORM_WORKER_THREADS` | 自动估算 | V8 backend platform worker 数 |
| `LEAPVM_TRACE_VM_THREAD_AFFINITY` | `0` | 输出 VM thread 亲和性调试信息 |
| `LEAPVM_SHUTDOWN_PLATFORM_ON_VM_THREAD` | `0` | 调整平台 shutdown 执行时机，排障用 |
| `LEAPVM_PREPARE_SHUTDOWN_AFTER_VM_THREAD` | `0` | 调整 prepare-shutdown 时机，排障用 |

### 测试 / manifest 辅助变量

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `LEAP_ENABLE_UNSTABLE_THREADPOOL_DOD` | `0` | 在 `full` manifest 中额外启用 `test-leapenv-threadpool-dod.js` |

## 2. 构建命令

### 构建 `leapvm_server`

```bash
cmake -S leap-vm -B leap-vm/build-server -DLEAPVM_BUILD_SERVER=ON
cmake --build leap-vm/build-server --target leapvm_server -j4
```

### 构建 bundle

```bash
cd leap-env
npm run build
```

### watch 模式

```bash
cd leap-env
npm run dev
```

## 3. 调试命令

### 单次执行

```bash
node run-work-leapvm.js --file h5st.js
```

### 带站点配置

```bash
node run-work-leapvm.js --file h5st.js --site-profile jd
node run-work-leapvm.js --file h5st.js --site-profile site-profiles/jd.json
```

### Inspector 调试

```bash
node run-work-leapvm.js --file h5st.js --debug --break --workers 1 --inspector-port 9229
```

建议：

- `--debug` 场景把 `requestTimeoutMs` 设为 `0`
- `workers` 设为 `1`

## 5. 说明

- `LEAP_SIGNATURE_PROFILE`、`fp-lean`、`fp-occupy` 已不是当前主线变量。
- 旧 ThreadPool / ProcessPool 参数不再作为主线文档入口；当前主线并发由服务端 WorkerPool 管理。
- Windows / macOS 与 Linux 的构建差异仍应视为补充主题，不纳入这里的主线命令表。
