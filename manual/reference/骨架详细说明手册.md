# 骨架详细说明手册

本文档只解释 skeleton 文件本身的字段、约束和示例；整体工作原理见 [Skeleton系统.md](/home/hostxxii/LeapWorkSpace/manual/architecture/Skeleton系统.md)。

## 1. 文件类型

当前 skeleton 分为两类：

| 类型 | 目录 | 命名规则 | 作用 |
|------|------|----------|------|
| type skeleton | `leap-env/src/skeleton/type/` | `Xxx.type.skeleton.js` | 描述类型模板、原型链和属性结构 |
| instance skeleton | `leap-env/src/skeleton/instance/` | `xxx.instance.skeleton.js` | 描述全局单例对象 |

所有文件都通过 IIFE 往 `leapenv.skeletonObjects[]` 追加对象。

## 2. 顶层字段

### type skeleton

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | `string` | 是 | 内部名，必须以 `.type` 结尾 |
| `ctorName` | `string` | 是 | 构造函数名 / V8 ClassName |
| `brand` | `string` | 是 | 品牌标签 |
| `ctorIllegal` | `boolean` | 否 | `new Xxx()` 是否抛错 |
| `exposeCtor` | `boolean` | 否 | 是否把构造函数暴露到全局 |
| `super` | `string \| null` | 是 | 父 type 的 `name` |
| `props` | `object` | 是 | 属性定义表 |

### instance skeleton

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | `string` | 是 | 内部名，必须以 `.instance` 结尾 |
| `instanceName` | `string` | 是 | 暴露到全局的属性名 |
| `brand` | `string` | 是 | 复用对应类型品牌 |
| `super` | `string \| null` | 是 | 继承的类型名或品牌链入口 |
| `ctorName` | `string` | 否 | 通常为空 |
| `exposeCtor` | `boolean` | 否 | 单例通常为 `false` |
| `ctorIllegal` | `boolean` | 否 | 单例通常为 `false` |
| `props` | `object` | 是 | 实例级直接属性定义 |

## 3. `props` 字段

每个属性项都有三类公共字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `owner` | `constructor \| prototype \| instance` | 挂载位置 |
| `kind` | `data \| method \| accessor` | 属性类型 |
| `brandCheck` | `boolean` | 是否启用非法调用校验 |
| `attributes` | object | `enumerable/configurable/writable` 标志 |

### `kind = data`

| 字段 | 类型 | 说明 |
|------|------|------|
| `valueType` | `string` | 数据值类型提示 |
| `value` | any | 静态值 |

### `kind = method`

| 字段 | 类型 | 说明 |
|------|------|------|
| `length` | `number` | `Function.length` |
| `dispatch` | object | `{ objName, propName }` |

### `kind = accessor`

| 字段 | 类型 | 说明 |
|------|------|------|
| `dispatch.getter` | object | getter 路由 |
| `dispatch.setter` | object | setter 路由 |

## 4. Symbol 编码

well-known symbol 属性在 skeleton 中使用字符串键编码：

| Symbol | 编码 |
|--------|------|
| `Symbol.toStringTag` | `@@toStringTag` |
| `Symbol.iterator` | `@@iterator` |
| `Symbol.toPrimitive` | `@@toPrimitive` |

这让 JS skeleton 能以普通对象字面量形式表达 symbol 属性，C++ 侧再做还原。

## 5. 示例

### type skeleton

```js
{
  "name": "Navigator.type",
  "ctorName": "Navigator",
  "brand": "Navigator",
  "ctorIllegal": true,
  "exposeCtor": true,
  "super": null,
  "props": {
    "userAgent": {
      "owner": "prototype",
      "kind": "accessor",
      "brandCheck": true,
      "attributes": { "enumerable": true, "configurable": true },
      "dispatch": {
        "getter": { "objName": "Navigator", "propName": "userAgent" }
      }
    }
  }
}
```

### instance skeleton

```js
{
  "name": "navigator.instance",
  "instanceName": "navigator",
  "brand": "Navigator",
  "super": "Navigator",
  "ctorName": "",
  "exposeCtor": false,
  "ctorIllegal": false,
  "props": {}
}
```

## 6. 约束

### IIFE 包装必须使用 `globalThis`

文件结尾必须是：

```js
})(globalThis);
```

不能写成 `})(this);`，否则打包成 IIFE 后顶层 `this` 可能不是全局对象。

### `window.instance` 必须最先加载

`window.instance.skeleton.js` 是唯一的 `window` 单例定义，构建时强制排在 instance skeleton 第一位。

### `super` 的含义

- type skeleton 的 `super` 用于原型继承
- instance skeleton 的 `super` 用于确定实例原型链与 `instanceof`

## 7. 当前边界

本手册不再包含：

- profile 过滤
- `fp-lean` / `fp-occupy`
- `filterSkeletonObjectsForProfile()`
- 历史 addon / NAPI 路线

这些都不是当前 standalone 主线的 skeleton 字段规范。
