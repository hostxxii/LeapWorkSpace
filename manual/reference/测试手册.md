# 测试手册

本文档描述当前 `tests/` 目录下的回归入口、manifest、脚本分类和 perf 基线文件。

## 1. 统一入口

```powershell
pwsh -File tests/runners/run-smoke.ps1
pwsh -File tests/runners/run-full.ps1
pwsh -File tests/runners/run-perf.ps1
pwsh -File tests/runners/run-manual.ps1
```

结果目录：

```text
tests/results/YYYYMMDD_HHMMSS_<profile>/
  summary.txt
  summary.json
  commands.log
  env.snapshot.txt
  outputs/
  perf-summary.md   # 仅 perf
```

## 2. Manifest 结构

### `smoke`

来源：`tests/manifest/smoke.ps1`

当前包含：

- `tests/scripts/leapvm/test_api.js`
- `tests/scripts/leapvm/test_timers.js`
- `tests/scripts/leapvm/test_globalthis.js`
- `tests/scripts/integration/test-leapenv-new-features.js`
- `tests/scripts/integration/test-leapenv-hook-isolation.js`

适用场景：快速确认 server 启动、基础 VM 能力和关键集成链路未断。

### `full`

来源：`tests/manifest/full.ps1`

当前主干覆盖：

- VM 基础：`test_api.js`、`test_timers.js`、`test_globalthis.js`
- DOM 主链：`test-leapenv-dom-minimal.js`、`test-leapenv-dom-m2-minimal.js`、`test-leapenv-dom-m3-minimal.js`
- DOM 原生追踪：`test-leapenv-dom-native-trace.js`、`test-leapenv-dom-native-ssot-consistency.js`
- 隔离与稳态：`test-leapenv-dom-pool-isolation.js`、`test-leapenv-dom-memory-leak.js`
- DoD：`test-leapenv-dod-layout.js`、`test-leapenv-dod-integration.js`、`test-leapenv-dod-converter.js`
- iframe：`test-leapenv-iframe.js`
- 并发：`test-leapenv-thread-pool-stability.js`
- 可选：`test-leapenv-threadpool-dod.js`（受 `LEAP_ENABLE_UNSTABLE_THREADPOOL_DOD` 控制）

适用场景：主线功能回归。

### `perf`

来源：`tests/manifest/perf.ps1`

当前包含：

- `tests/scripts/perf/check-thread-mainline-gate.js`
- `tests/scripts/perf/bench-thread-vs-process.js`
- `tests/scripts/perf/bench-process-pool.js`
- `tests/scripts/perf/bench-dom-multi-scale.js`
- `tests/scripts/perf/bench-dom-final-shape-baseline.js`

适用场景：吞吐、延迟、内存和对比基线观测。

## 3. 脚本目录

### `tests/scripts/integration/`

当前主要类别：

- 运行时与全局面：`test-leapenv-global-surface.js`、`test-leapenv-hardening-modes.js`
- 指纹与签名：`test-leapenv-fingerprint-snapshot.js`、`test-leapenv-signature-core.js`
- DOM / BOM：`test-leapenv-dom-*`
- 特殊 API：`test-leapenv-crypto-minimal.js`、`test-leapenv-canvas-minimal.js`
- iframe：`test-leapenv-iframe.js`
- 并发与 worker：`test-leapenv-thread-pool-stability.js`、`test-leapenv-worker-threads.js`
- code cache / 构建辅助：`bench-code-cache.js`、`check-entry-coverage.js`

### `tests/scripts/leapvm/`

偏底层 VM / server 验证：

- `test_api.js`
- `test_timers.js`
- `test_globalthis.js`
- `test_shutdown.js`
- `test_inspect_brk.js`
- `test_dom_native_parse.js`
- `test_highres.js`

### `tests/scripts/perf/`

性能与门禁：

- `check-thread-mainline-gate.js`
- `bench-thread-vs-process.js`
- `bench-process-pool.js`
- `bench-dom-multi-scale.js`
- `bench-dom-final-shape-baseline.js`
- `bench-threadpool-dod.js`
- `stress-test-threadpool-dod.js`

### `tests/scripts/manual/`

需要人工判断或 DevTools 交互的脚本与说明。

## 4. Perf 基线文件

| 文件 | 作用 |
|------|------|
| `tests/baselines/perf-baseline.md` | 历史样本与关键性能结果 |
| `tests/baselines/thresholds.md` | perf 观察阈值与门禁说明 |

## 5. 常用单脚本入口

```bash
node tests/scripts/integration/test-leapenv-dom-minimal.js
node tests/scripts/integration/test-leapenv-iframe.js
node tests/scripts/integration/test-leapenv-crypto-minimal.js
node tests/scripts/integration/test-leapenv-canvas-minimal.js
node tests/scripts/integration/bench-code-cache.js
node tests/scripts/leapvm/test_inspect_brk.js
```

## 6. 维护原则

- `smoke` 只放最短主链路。
- `full` 放当前主线模块回归，不混历史 addon 路线。
- `perf` 只放可重复、可对比的脚本。
- 需要人工接入 DevTools 的项放 `manual/` 或单独说明，不伪装成自动化通过项。
