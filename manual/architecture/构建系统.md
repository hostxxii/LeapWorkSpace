# 构建系统

本文档描述 leap-env bundle 的生成过程和 leapvm_server 的 CMake 主线构建目标。

## 1. leap-env bundle 构建

### 入口命令

```bash
cd leap-env && npm run build
```

执行 `src/build/build.js`，内部流程：

1. **generate-entry.js** → 生成 `src/build/entry.js`
2. **validate-skeleton-context.js** → 校验 skeleton 文件
3. **esbuild** → 打包为 `src/build/dist/leap.bundle.js`

### generate-entry.js

自动扫描 `src/` 目录，按可控顺序生成 entry.js。

#### 加载分组与顺序

```
1. core/            CORE_ORDER 写死顺序
2. skeleton/type/   按继承拓扑排序
3. skeleton/instance/  window.instance.skeleton.js 排首
4. skeleton/other/  字典序
5. impl/            字典序
6. instance/        字典序
7. other/           字典序
```

#### CORE_ORDER

```javascript
const CORE_ORDER = [
  'runtime.js',
  'config.js',
  'tools.js',
  'skeleton-loader.js'
];
```

顺序不可变：runtime 建立 namespace → config 读取 bootstrap → tools 注册工具 → skeleton-loader 加载 skeleton。

#### skeleton/type 拓扑排序

使用 Kahn 算法：

1. 从每个 type skeleton 文件中提取 `"name"` 和 `"super"` 字段
2. 构建依赖 DAG（parent → child）
3. BFS 拓扑排序，父类先于子类
4. 检测到循环依赖 → 警告 + 回退到字典序

#### skeleton/instance 排序

`window.instance.skeleton.js` 强制排在所有 instance skeleton 之前。其余字典序。

#### 排除目录

- `src/build/`：生成脚本与产物（自身）
- `src/pool/`：已废弃的并发池，不进入 VM bundle
- `src/client/`：Node.js 侧 TCP 客户端，宿主模块

### validate-skeleton-context.js

扫描 `src/skeleton/` 下所有 `.js` 文件，检查是否存在 `})(this);` 结尾。

- 所有 skeleton IIFE 必须以 `})(globalThis);` 结尾
- `})(this);` 在 esbuild IIFE 模式下会导致 `this` 绑定为 `exports` 而非全局对象
- 发现违规 → 打印位置 → 构建失败

### esbuild 配置

```javascript
{
  entryPoints: ['src/build/entry.js'],
  bundle: true,
  outfile: 'src/build/dist/leap.bundle.js',
  format: 'iife',
  platform: 'neutral',
  target: 'es2020',
  minify: false,
  sourcemap: true
}
```

| 选项 | 说明 |
|------|------|
| `format: 'iife'` | 立即执行函数，在 V8 Context 内直接执行 |
| `platform: 'neutral'` | 不引入 Node.js 或浏览器特定 polyfill |
| `minify: false` | 开发阶段不压缩 |
| `sourcemap: true` | 生成 source map 用于 Inspector 调试 |

### watch 模式

```bash
cd leap-env && npm run dev
```

等同于 `node src/build/build.js --watch`，文件变更自动重新打包。

注意：watch 模式不会重新扫描新增文件。添加新 `.js` 文件后需重启构建。

### 产物

```
leap-env/src/build/dist/
├── leap.bundle.js          # bundle 主文件
└── leap.bundle.js.map      # source map
```

## 2. leapvm_server CMake 构建

### 构建命令

```bash
cmake -S leap-vm -B leap-vm/build-server -DLEAPVM_BUILD_SERVER=ON
cmake --build leap-vm/build-server --target leapvm_server -j4
```

### 主要选项

| 选项 | 默认 | 说明 |
|------|------|------|
| `LEAPVM_BUILD_SERVER` | OFF | 构建 standalone server 二进制 |
| `LEAPVM_ENABLE_LEXBOR` | ON | 启用 Lexbor HTML 解析器 |

### 服务层源文件

当 `LEAPVM_BUILD_SERVER=ON` 时编译：

```
leap-vm/src/service/
├── main.cc              # 入口
├── worker_pool.cc/h     # Worker 线程池
├── ipc_server.cc/h      # TCP IPC 服务
├── task_protocol.cc/h   # 任务脚本生成
└── config_loader.cc/h   # 文件加载
```

加上 `src/leapvm/` 下的 VM 核心：

```
vm_instance.cc       # V8 Isolate/Context 生命周期
dom_core.cc          # Skeleton Builder / Dispatch Bridge
monitor.cc           # Hook/Monitor
hook_filter.cc       # Hook 过滤规则
v8_platform.cc       # V8 平台初始化
...
```

### 产物

```
leap-vm/build-server/leapvm_server    # standalone 可执行文件
```

### 平台差异

- **Linux**：V8 Clang + libc++ + ThinLTO
- **Windows**：独立链接与编译选项分支
